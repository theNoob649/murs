Snapshots & Previews

How to play a game drop months before it comes out — and what you're signing up for.

The Testing Pipeline

Every drop goes through the same stages before release. On Java the sequence is snapshots → pre-releases → release candidates → release. On Bedrock, a parallel Preview (or Beta) build runs alongside the retail version the whole time.

These builds are the main way features get revealed between Minecraft Live shows. When a drop enters snapshots, Mojang starts publishing what's in it week by week.

Java Edition: Snapshots

Snapshots are in-development builds, released roughly weekly, playable through the official launcher.

The old naming scheme (used through 2025) was YYwWWn: two-digit year, the letter w, the ISO week number, then a letter for the order within that week. 25w14a was the first snapshot of week 14 of 2025. It told you exactly when a build came out — but nothing about which update it belonged to.

The new scheme (2026 onward) is YY.D-snapshot-N — for example 26.3-snapshot-1. The target drop is now part of the name. Mojang's own example: the first Mounts of Mayhem snapshot shipped as 25w41a, which under the new system would have been 25.4-snapshot-1.

Pre-releases come after the snapshot cycle closes. The feature set is frozen; only bugs get fixed. Named like 1.14.1 Pre-Release 2.

Release candidates are meant to be the final build unless something game-breaking turns up. Named like 1.16-rc1. The record is 1.18, which needed four.

How to Opt In (Java)

Open the Minecraft Launcher → Installations → New installation → tick Snapshots in the version filter → pick the snapshot you want. It installs alongside your normal version, so you don't lose your release install.

Experiments & Feature Flags (Java)

Java has no in-game experiments menu. Unreleased content is gated behind feature flags chosen at world creation: on the world-creation screen, click Experiments and enable the relevant built-in datapack.

Enabling an experiment marks that world as experimental permanently. You get a warning, and it cannot be cleanly undone.

Bedrock Edition: Preview & Beta

Minecraft Preview is a separate application that installs alongside your retail copy. Available on Windows, Xbox, PlayStation 4/5, and iOS/iPadOS (via TestFlight, where slots are limited and free up at the start of each month). You need to own Bedrock Edition, or have Game Pass on Windows/Xbox.

Preview builds show debug info on screen, have achievements and trophies disabled, and can't connect to retail servers. They carry their own version stream that runs ahead of retail.

Bedrock Beta is the same thing under a different delivery method — on Android you opt in through the Google Play beta program. Unlike Preview, the Android beta replaces your retail install, so back up your worlds first.

Experimental Toggles (Bedrock)

Create New World → Game settings → Experiments → enable what you want → confirm. Existing worlds can do the same from their settings; the game automatically saves a "Copy of <world>" backup first.

There are two categories: Gameplay toggles for unreleased player-facing features, and Add-On Creators toggles for scripting and API surfaces.

Before You Do Any of This

Back up your worlds. This is not boilerplate. Snapshot and preview builds can corrupt saves, and a world opened in a newer build usually cannot be opened in an older one again.

Turning on experiments is permanent. On Bedrock it disables achievements for that world forever. On both editions the world is flagged and cannot be reverted.

Features get cut. Anything in a snapshot can be changed or removed before release — fireflies were announced for The Wild Update and never shipped. Don't build a base around something that isn't out yet.
