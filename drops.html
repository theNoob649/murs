How the Drop System Works

What Is a Game Drop?

For over a decade, Minecraft shipped one big free update per year. Caves & Cliffs. The Wild Update. Trails & Tales. Tricky Trials. Each got a name, a Minecraft Live reveal, and then a long wait — a feature announced in October might not reach players until the following June.

Mojang scrapped that. A game drop is a smaller, named, themed update shipped on a roughly quarterly cadence — three to four per year. Instead of holding finished features hostage to a yearly release date, they go out when they're done.

What Changed

Minecraft Live happens more than once a year. Shows are now scheduled near drop windows rather than as a single annual event. 2026 had one in March and a bonus show in May.

The mob vote is gone. Mojang discontinued it after years of community frustration. Mobs now simply ship when they're built.

Big updates aren't dead. Mojang has said the drop model doesn't rule out occasional larger, transformative updates layered on top.

The New Version Numbers

In December 2025, Mounts of Mayhem became the last release to use 1.x.y — the numbering Minecraft had used since 2011.

The format is now YY.D: the year, then which drop of that year it is. 26.1 is the first drop of 2026, 26.2 the second. A third number (26.1.1) means a bug-fix patch, not a drop.

Java and Bedrock share the year prefix but diverge after it, because Bedrock ships more intermediate builds — the same drop is 26.1 on Java and 26.10 on Bedrock. The stated goal was to let server admins, modders, and creators tell at a glance whether a number is a drop or a patch.
